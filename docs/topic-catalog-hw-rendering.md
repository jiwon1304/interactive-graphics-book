# GPU 하드웨어 ↔ 렌더링 접점 토픽 카탈로그 (집필 백로그)

"GPU 실행 모델이 렌더링에서 어떻게 도는가"를 중심으로, 메모리/대역폭·텍스처 필터링/밉매핑·텍스처 압축·
타일 기반/모바일 GPU까지 — **하드웨어와 렌더링이 만나는 지점**. 순수 렌더링 알고리즘([topic-catalog.md](./topic-catalog.md))이나
순수 하드웨어([topic-catalog-gpu.md](./topic-catalog-gpu.md))와 구분되는 교차 영역. 병렬 웹 리서치(2026-06)·중복 제거.

**도식 규칙(중요):** *효과가 시각적*(앨리어싱·필터 품질·압축 아티팩트·이방성·오버드로)이면 **인터랙티브**,
*하드웨어 메커니즘/데이터플로*(쿼드 실행·early-Z·DCC·타일 비닝·포맷 구조)면 **정적 도식**. 도식 안 글자는 최소, 설명은 캡션.
표기: `[난이도] 한글명 (English) — 데모 힌트 (I=인터랙티브 / S=정적)`.

---

## A. 렌더링에서의 GPU 실행 모델 (셰이딩이 실제로 어떻게 도는가)
1. `[입문]` 2×2 픽셀 쿼드 (Pixel quads) — 삼각형이 닿은 2×2 쿼드 전체가 켜짐; 작을수록 낭비 (I)
2. `[중급]` 헬퍼 레인 & 쿼드 오버셰이딩 (Helper lanes) — 가장자리 쿼드 25~75% 낭비, 헬퍼 비율 카운터 (I)
3. `[중급]` ddx/ddy = 쿼드 차분 (Screen-space derivatives) — 이웃 픽셀 유한차분, "우−나=ddx" (I)
4. `[심화]` coarse vs fine 미분 — 쿼드당 1차분 공유(D3D 기본) vs 픽셀별 (S)
5. `[중급]` 미분 → 밉 LOD 선택 (LOD from quad gradients) — UV 미분→텍셀 footprint→밉 레벨 (I) [→B와 연결]
6. `[심화]` 워프/웨이브 픽셀 셰이더 다이버전스 — 32(NV)/64(AMD) 락스텝, 분기 섞이면 비용 2배 (I)
7. `[심화]` 픽셀 셰이더 쿼드/웨이브 intrinsics (QuadReadAcross) — 레인 간 셔플 경로 (S)
8. `[중급]` Early-Z vs Late-Z — 깊이 테스트 앞/뒤, 픽셀 셰이더 호출 수 변화 (I)
9. `[심화]` discard·알파테스트·SV_Depth가 early-Z를 끄는 이유 — 의사결정 흐름도 (S)
10. `[중급]` Hi-Z / 계층적 깊이 컬링 — 타일 min/max로 삼각형 타일 단위 통과/기각 (I) [→D]
11. `[중급]` 오버드로우 & 깊이 프리패스 — 오버드로 히트맵 + 프리패스 토글 (I)
12. `[중급]` 원근 보정 보간 — 1/w 공간 선형, 선형보간 시 텍스처 워핑 결함 (I)
13. `[심화]` 정점 재사용: 포스트-트랜스폼 캐시 (Vertex reuse) — 배치 단위(≈32삼각형) 재사용, ACMR (I)
14. `[심화]` 삼각형 셋업·비닝·래스터 분배 — 에지 함수→스크린 타일 비닝→GPC 병렬 (S)
15. `[심화]` 픽셀 셰이딩 어큐펀시 & 레이턴시 은닉 — VGPR 압력↑ → 동시 웨이브↓ → stall (I)

## B. 텍스처 필터링 & 밉매핑
16. `[입문]` 미니피케이션 앨리어싱 — 멀리서 모아레·shimmer, 필터 off일 때 살아 움직임 (I)
17. `[입문]` 나이퀴스트 & 사전 필터링 — 밉=샘플링 전 대역제한, 1D 앨리어싱 도식 (S)
18. `[입문]` 밉 체인 (1/2 피라미드) — 레벨당 1/4, 전체 ≈4/3(+33%) 메모리 (I)
19. `[중급]` LOD 선택 (ddx/ddy) — $\lambda=\log_2\rho$, 텍셀 footprint 평행사변형 (I)
20. `[입문]` 양선형 보간 (Bilinear) — 4 텍셀 가중치, 샘플점 드래그 (I)
21. `[중급]` 삼선형 보간 (Trilinear) — 두 밉 사이 LOD 소수부 블렌드, 밉 경계 띠 제거 (I)
22. `[심화]` 이방성 필터링 — footprint 타원, 장축 따라 N탭, max-aniso 1~16 (I)
23. `[심화]` AF 비용 트레이드오프 — 탭 수↔비용↔선명도 막대 (S)
24. `[심화]` 텍스처 캐시 & 지역성 — 작은 L1(4~8KB), 밉으로 1:1 유지 시 미스↓ (S)
25. `[중급]` 확대 필터 & 8비트 서브텍셀 정밀도 — nearest↔bilinear, 256단계 밴딩 (I)
26. `[중급]` 랩/클램프/미러 주소 지정 — UV 범위 밖 처리 모드별 타일 (I)
27. `[심화]` 밉 바이어스 & 음수 LOD 샤프닝 — TAA 복원 vs shimmer 트레이드오프 (I)
28. `[심화]` 감마-정확 밉 생성 — sRGB 선형화 후 필터, box vs Kaiser (I)
29. `[심화]` sRGB 필터링 — 선형 공간에서 블렌드, 순서 뒤집으면 너무 어두움 (I)
30. `[최신]` 분기하 LOD undefined — 쿼드 분기 시 미분 깨짐, textureGrad/Lod 우회 (S)

## C. 텍스처 압축 (블록 압축)
31. `[입문]` 왜 압축하나 — 32-bit 대비 최대 75% 절감(BC1 6:1), VRAM/대역폭 막대 (I)
32. `[입문]` BC1/DXT1 해부 — 4×4=2 엔드포인트(RGB565)+2-bit 인덱스=8B, 선분 위 투영 (I)
33. `[입문]` 블록 아티팩트 & 녹색 밴딩 — 그라데이션에 4×4 경계, 565 비대칭 (I)
34. `[중급]` BC1 펀치스루 알파 — 엔드포인트 순서가 모드 플래그(투명 1-bit) (I)
35. `[중급]` BC3/DXT5 알파 블록 — 8-bit 엔드포인트+3-bit 인덱스=8값 팔레트 (I)
36. `[중급]` BC4/BC5 단일·이중 채널 — 높이맵/러프니스, 독립 2채널 8값 (I)
37. `[중급]` 노멀맵 압축: BC5 + Z 재구성 — $z=\sqrt{1-x^2-y^2}$, BC1 녹색틴트와 대비 (I)
38. `[심화]` BC7/BPTC — 8모드·파티션(64종)·P-bit로 블록별 적응 (S 파티션 갤러리 + I 품질)
39. `[심화]` BC6H HDR — 유일 네이티브 HDR, FP16 16B, 14모드 (I 사전인코딩 토글)
40. `[심화]` ASTC — 128-bit 고정 × 가변 footprint(4×4~12×12) = 8.0~0.89bpp 연속 (I)
41. `[중급]` ETC2/EAC — 모바일 기본, T/H/Planar 모드로 ETC1 보완 (I 모드 토글)
42. `[중급]` 하드웨어 디코드(TMU) — 압축 상태로 VRAM/L2 통과, 샘플 시 즉석 디코드 (S)
43. `[중급]` 채널 패킹 (ORM) — AO·러프·메탈 3장을 RGB 한 장에, 샘플러 1/3 (I)
44. `[최신]` 슈퍼컴프레션: Basis/KTX2/UASTC — GPU 포맷 위 2차 압축 + 런타임 트랜스코드 (S)
45. `[최신]` 웹의 압축 텍스처: KTX2Loader — 기기 포맷 감지→트랜스코드, "한 에셋 모든 GPU" (I capability 탐침)

## D. 메모리 대역폭 & 프레임버퍼/Z 압축
46. `[입문]` 대역폭이 진짜 병목 — 연산≫대역폭, compute/bandwidth-bound 막대 (I)
47. `[중급]` 루프라인 모델 — 산술강도↔성능, ridge point에서 병목 전환 (I)
48. `[입문]` 대역폭 예산 계산 — 해상도×bpp×(읽기+쓰기+오버드로)×fps vs GPU GB/s (I)
49. `[중급]` 델타 컬러 압축 (DCC) — 블록당 anchor+delta, 컬러 트래픽 30~70%↓ (S)
50. `[중급]` 무손실 압축의 대역폭 증폭 — 그라데이션 vs 노이즈 블록 압축률 히트맵 (I)
51. `[심화]` Z/깊이 압축 — 평면 predictor + residual(Golomb-Rice) (S)
52. `[입문]` Fast clear (메타데이터) — 8.3MB 쓰기 대신 수KB 메타, 100~1000× (S)
53. `[심화]` MSAA HW: coverage/색 분리 + FMASK — 샘플↔색조각 매핑 테이블 (S)
54. `[중급]` ROP / 픽셀 백엔드 — depth test→blend→compress→write, 대역폭 진원지 (S)
55. `[중급]` 메모리 스위즐 & Morton/Z-order — 좌표 비트 인터리브로 2D 캐시 지역성 (I)
56. `[중급]` 텍스처/L2 캐시 적중 — 접근 패턴↔적중률, 디컴프는 L1–L2 사이 (I)
57. `[심화]` 채널/뱅크 인터리빙 — 주소→채널 분산(spray), 캠핑 시 대역폭 폭락 (S)
58. `[최신]` 표시 가능 DCC & 압축 한계 — 압축 표면 직접 스캔아웃, RTT no-op (S)

## E. 타일 기반 렌더링 & 모바일 GPU
59. `[입문]` 왜 모바일은 타일로 그리나 — IMR vs TBR, DRAM 쓰기 횟수 카운터 (I)
60. `[입문]` 온칩 타일 메모리(GMEM) — 16×16~32×32 타일이 1~5MB SRAM에, 끝날 때만 DRAM (S)
61. `[중급]` 비닝 패스 + 렌더 패스 — 버텍스 먼저 → 타일별 프리미티브 리스트 → 픽셀 (I)
62. `[중급]` IMR vs TBDR 대역폭 — 외부 트래픽 ≈1.96× 절감, 슬라이더 계산기 (I)
63. `[중급]` HSR: 숨은 면 제거, 제로 오버드로 — 보이는 픽셀만 셰이딩(모바일선 z-prepass 역효과) (I)
64. `[중급]` 모바일에서 오버드로가 비싼 이유 — 반투명이 early-ZS 끔, 겹칠수록 비용↑ (I)
65. `[중급]` 렌더 패스 load/store op — loadOp/storeOp가 타일↔DRAM 화살표 제어 (S)
66. `[심화]` 서브패스 온타일 디퍼드 — input attachment로 G-buffer를 타일에, DRAM 왕복 0 (S)
67. `[심화]` 픽셀 로컬 스토리지 / framebuffer fetch — 같은 픽셀 이전값 타일에서 read-modify-write (S)
68. `[중급]` 타일에서 거의 공짜인 MSAA — 샘플을 타일에 두고 write-back 시 resolve (I)
69. `[심화]` Adreno FlexRender & LRZ — binned↔direct 동적 전환, 저해상도 Z 조기 기각 (S)
70. `[심화]` Apple/Metal TBDR: imageblock·tile shader·memoryless — 타일 메모리 명시 노출 (S)
71. `[심화]` 파라미터 버퍼 & 지오메트리 폭증 — 비닝의 대가, 버퍼 넘치면 부분 플러시 (I)
72. `[중급]` HSR를 깨뜨리는 것: discard & 제출 순서 — 불투명→알파테스트→블렌드 황금률 (I)
73. `[심화]` 대역폭이 곧 전력 — DRAM ~60~150pJ/byte vs 연산 ~50fJ, 에너지 비대칭 (I)
