# 모바일 퍼스트 Figure 작성 지침 (MOBILE-FIGURES)

> 이 책은 **대부분 모바일(세로, 좁은 화면)에서 읽힌다.** 모든 figure/데모는 모바일에서 먼저
> 잘 보이도록 설계한다. 데스크톱은 여유가 있으니 모바일이 통과하면 자동으로 통과한다.

## 핵심 사실 (왜 이렇게 하나)
- 본문 컬럼은 모바일에서 **화면 폭 전체**(대략 CSS 360–414px, 패딩 빼면 ~330–390px)를 쓴다.
  설계 기준 폭은 **360px**로 잡는다.
- `<canvas>`는 `width:100%; height:auto`로 **하나의 비트맵이 통째로 축소**된다. 즉 캔버스 내부에
  px로 그린 글자는 화면에서 `onScreen = fontPx × (표시폭 / 캔버스내부폭)`로 작아진다.
  → 내부 폭이 넓고 표시폭이 좁을수록 글자가 깨알처럼 작아진다.
- 모바일 가독성 최소 글자 크기는 **화면상 ≈ 11px**. iOS Safari는 가로 스크롤/줌을 유발하는 넓은
  요소를 싫어한다(특히 `<table>`, 긴 타임라인).

## 캔버스 사이징 규칙 (제일 중요)
1. **내부 폭을 좁게.** 360px 화면(표시 ~340px)에서 글자가 11px 이상 보이려면:
   `fontPx ≥ 11 × (캔버스내부폭 / 340)`.
   - 내부폭 **360** → 12px 글자도 화면상 ~11px (OK). **권장: 내부폭 ≤ 약 400.**
   - 내부폭 560 + 12px 글자 → 화면상 ~7px (❌ 너무 작다). 560 같은 넓은 캔버스 금지.
2. **글자는 최소 12–13px(내부 px)**, 제목/수치는 14–15px. 보조 라벨도 11px 미만 금지.
3. **세로로 키워라.** 모바일은 세로 스크롤이 자연스럽다. 가로로 넓게 늘어놓지 말고
   **위→아래 스택**으로 배치(타임라인·다단 비교·플로우 다이어그램 모두). 세로로 긴 figure는 OK.
4. 종횡비는 모바일에서 **세로로 너무 길지 않게**(한 화면에 맥락이 보이게, 대략 높이 ≤ 폭×1.6 권장).
5. 라벨은 **짧게**(약어·아이콘), 밀도 낮게. 길면 줄이거나 figcaption으로 옮긴다.
6. 캔버스 스타일: `width:100%; height:auto; max-width:<내부폭>px` + 모서리/테두리. 절대 고정 px 폭 금지.

## DOM 기반 figure(박스 다이어그램 등)
- flex/grid는 **`flex-wrap`** + 자식에 **`min-width:0`**(안 그러면 안 줄어듦) + **`overflow-wrap:anywhere`**.
- 모든 박스·셀에 텍스트가 **박스를 넘지 않게**: `overflow-wrap/word-break`, 패딩 확보, 폰트 상대크기(`clamp()` 권장).
- 미디어/`svg`/`canvas`에 `max-width:100%`. 가로 스크롤 유발 금지(넓은 표는 스택·축약).
- 인터랙션이 남아 있으면 탭 타깃 ≥ 44×44px.

## 정적(static) 도식 원칙 (이 책의 현재 방향)
- 렌더링(3D 장면·셰이딩·텍스처·지오메트리)이 **아닌** 개념도/차트/타임라인/플로우/계산기/카운터는
  **인터랙션 없이 정적 1컷**으로. 가장 설명력 있는 한 상태를 골라 그린다.
- 정적이라도 위 모바일 규칙(좁은 내부폭·큰 글자·세로 스택·박스 넘침 방지)을 똑같이 지킨다.
- figcaption에 "직접 해보세요" 대신 그림이 보여주는 핵심을 한 문단으로 설명한다.

## 체크리스트 (배포 전)
- [ ] 360px 폭에서 모든 글자가 읽힌다(화면상 ≥ 11px).
- [ ] 가로 스크롤이 없다(figure가 화면 폭을 넘지 않음).
- [ ] 박스 안 글자가 박스를 벗어나지 않는다.
- [ ] 캔버스 내부폭 ≤ ~400, 글자 ≥ 12px.
- [ ] 라이트/다크 양쪽에서 대비가 충분하다(CSS 변수 토큰 사용).
